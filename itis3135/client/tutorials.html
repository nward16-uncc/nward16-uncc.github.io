Tutorials

This walkthrough shows a methodology for approaching your first HTB machine; Lame. Work through each step yourself before using the hints or answers.

Step 1 — Reconnaissance

Start with a basic port and service scan to discover open ports and services.

nmap -sC -sV -oA nmap/initial 10.10.10.123

Look for common services such as FTP, HTTP, SSH, and note versions and banners.

Step 2 — Service enumeration

Dig deeper into services found during the scan. For web services, try a directory scan. For FTP, check for anonymous access or interesting files.

gobuster dir -u http://10.10.10.123 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt

Record filenames, versions, and any credentials you find; these are often the foothold.

Step 3 — Initial Access

Use the information from enumeration to attempt an initial login or exploit. Try non-destructive approaches first, such as logging in with discovered credentials or downloading interesting files for further analysis.

Step 4 — Privilege Escalation

After a foothold, collect system information and search for local privilege escalation paths. Useful checks include kernel version, sudo privileges, and SUID binaries.

uname -a
id
sudo -l
ls -la /home
